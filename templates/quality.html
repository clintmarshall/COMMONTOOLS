Qualifier — Quality Dashboard

Overall Quality Score

Maintainability Index

How easy the code is to maintain. Higher is better. Target: 80+. Below 60 means the code is hard to work with.

Statement Coverage

What percentage of code is exercised by tests. Higher means more confidence that changes won't break things. Target: 80%+.

Branch Coverage

How many if/else paths are tested. Low branch coverage means some code paths have never been tested.

Function Coverage

What percentage of functions have been called by tests. Every function should be tested at least once.

Line Coverage

What percentage of lines run during tests. Similar to statement coverage but counts individual lines.

Dead Files %

Files that do nothing useful — no imports, no exports used. Dead files add confusion and slow down builds.

Duplication %

Code that's been copy-pasted. Duplicated code means bugs need fixing in multiple places. Target: under 10%.

Max CRAP — App Code

Complexity Risk Anti-Pattern. Measures the most complex function. High CRAP = hard to change without breaking things.

Test Files

Number of test files found. More test files generally means broader coverage.
